## Stack

TanStack Start (file-based routing) + React 19 + Vite 8 + Supabase (auth + Postgres).
UI: shadcn/ui (new-york style) + Tailwind CSS v4. Package manager: **npm**.

## Commands

```bash
npm install
npm run dev        # Vite dev server
npm run build      # production build
npm run lint       # eslint .
npm run format     # prettier --write .
```

There is no `typecheck` or `test` script. After edits, run `npm run lint` to catch issues.

## Routing

File-based via `src/routes/`. `routeTree.gen.ts` is auto-generated — never edit by hand.
Root layout: `src/routes/__root.tsx`. Auth guard layout: `src/routes/_authenticated/route.tsx`.

TanStack Start conventions (not Next.js):
- Dynamic param: `$id` (not `{id}` or `[id]`)
- Optional segment: `{-$category}`
- Splat: `$.tsx`, read via `_splat`
- Layout route: `_layout.tsx` with `<Outlet />`

## Supabase

- **Client** (browser/SSR): `@/integrations/supabase/client` — uses `VITE_*` env vars, lazy singleton.
- **Admin** (server-only, bypasses RLS): `@/integrations/supabase/client.server` — uses `SUPABASE_SERVICE_ROLE_KEY`. Dynamic-import in server handlers; top-level import only in `.server.ts` files.
- **Auth middleware**: `auth-middleware.ts` validates Bearer tokens in server functions; `auth-attacher.ts` (registered globally in `src/start.ts`) attaches tokens from the browser session.
- `.env` contains the publishable (anon) key only. Service role key is not committed.

## Auth flow

`_authenticated/route.tsx` runs `supabase.auth.getUser()` in `beforeLoad` — redirects to `/auth` if unauthenticated. All routes under `_authenticated/` inherit this guard.

## Vite config

Uses `@lovable.dev/vite-tanstack-config` which bundles TanStack Start, React, Tailwind, tsconfig paths, nitro (Cloudflare target), and env injection. Do **not** add these plugins manually — it will break with duplicates.

## Style

- Prettier: 100 print width, double quotes, trailing commas, semicolons.
- Path alias: `@/*` → `./src/*`.
- ESLint bans `server-only` imports (use `*.server.ts` suffix instead).
- Custom categories are stored in `localStorage` (key `vault:custom_categories_v2`), not in the database.

## Gotchas

- `src/integrations/supabase/client.ts` and `client.server.ts` are auto-generated by Lovable — do not edit directly.
- `src/server.ts` wraps the TanStack Start server entry with SSR error normalization (h3 swallows errors into opaque 500 JSON).
